## 1. 线性研究流程

**适用场景：** 步骤有明确依赖关系，必须按顺序执行

```
Input → 研究员 → 分析师 → 写作助手 → Output
```

**关键配置：**
- 研究员：web_search + url_fetch，maxIterations: 15
- 分析师：无需工具，依赖研究员的输出，maxIterations: 10
- 写作助手：无需工具，maxIterations: 8

**节点位置：**
- Input: x=100, y=300
- 研究员: x=400, y=300
- 分析师: x=700, y=300
- 写作助手: x=1000, y=300
- Output: x=1300, y=300

## 2. 并行研究 + 汇总

**适用场景：** 多个独立的研究方向，可以同时进行

```
Input ──→ 研究员A ──→┐
       ├→ 研究员B ──→ Merge → 分析师 → Output
       └→ 研究员C ──→┘
```

**关键配置：**
- 三个研究员并行，各自独立的 systemPrompt 和研究方向
- Merge 节点等待所有上游完成后汇总
- 分析师拿到汇总内容后做综合分析

**节点位置：**
- Input: x=100, y=400
- 研究员A: x=400, y=200
- 研究员B: x=400, y=400
- 研究员C: x=400, y=600
- Merge: x=700, y=400
- 分析师: x=1000, y=400
- Output: x=1300, y=400

## 3. 质量门控循环

**适用场景：** 输出必须达到一定质量标准，否则重做

```
Input → 执行者 → 质检员 → Condition ──true──→ Output
                              └──false──→ 执行者（循环）
```

**关键配置：**
- 执行者：完成核心任务
- 质检员：评估输出质量，输出结构化评估结果
  systemPrompt 要说明：输出 JSON `{"passed": true/false, "feedback": "..."}`
- Condition：自然语言模式，条件写"如果质检通过（passed 为 true）"
- maxLoopIterations: 3（防止无限循环）

**质检员 systemPrompt 模板：**
```
你是严格的质量检查员。评估上游 Agent 的输出是否满足以下标准：
[标准1]
[标准2]
[标准3]

输出 JSON 格式：
{
  "passed": true 或 false,
  "score": 1-10 的评分,
  "feedback": "具体的改进建议（如果 passed 为 false）"
}
```

## 4. 条件分支路由

**适用场景：** 根据输入内容走不同的处理路径

```
Input → 分类器 → Condition ──true──→ 路径A → Output
                    └──false──→ 路径B → Output
```

**关键配置：**
- 分类器：判断输入属于哪类，输出分类结果
- Condition：表达式模式效果更稳定
  例如：`output.category === "technical"`
- 两条路径分别处理不同类型的输入

## 5. 分层处理流程

**适用场景：** 复杂任务需要先拆解再执行

```
Input → 规划师 → 执行者A ──→┐
                ├→ 执行者B ──→ Merge → 整合者 → Output
                └→ 执行者C ──→┘
```

**关键配置：**
- 规划师：不需要工具，专注于任务分解
  systemPrompt 说明：输出结构化的任务列表
- 执行者：各自承接规划师分配的子任务
- 整合者：综合所有执行结果，输出最终成果

---

## 6. 多角度评估

**适用场景：** 需要从多个专业视角评估同一件事

```
Input ──→ 技术评审员 ──→┐
       ├→ 商业评审员 ──→ Merge → 综合评估师 → Output
       └→ 用户体验员 ──→┘
```

**关键配置：**
- 三个评审员从不同维度评估同一输入
- 每个评审员的 systemPrompt 强调其专业视角
- 综合评估师权衡三方意见，给出最终建议

## 场景速查

| 用户需求 | 推荐模式 | 节点数量 |
|---------|---------|---------|
| 竞品分析 | 并行研究 + 汇总 | 6-8 |
| 内容创作 | 线性流程 | 4-5 |
| 代码开发 | 质量门控循环 | 5-6 |
| 市场调研 | 并行研究 + 汇总 | 7-9 |
| 数据分析 | 分层处理 | 5-7 |
| 方案评估 | 多角度评估 | 6-8 |
| 文档处理 | 线性流程 | 3-4 |
| 问答系统 | 条件分支 | 4-6 |
